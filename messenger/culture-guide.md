# 앵커 사업단 Slack 커뮤니케이션 문화 가이드

## 기본 원칙

앵커 사업단 Slack은 빠른 소통과 협업을 위한 도구입니다.  
효율적이고 존중하는 커뮤니케이션 문화를 함께 만들어 나갑니다.

---

## 메시지 작성 원칙

### 명확하고 간결하게

- 결론 먼저, 배경은 나중
- 긴 내용은 스레드 또는 문서로 정리 후 링크 공유
- 이모지는 맥락에 맞게 절제해서 사용

### 수신자를 배려하는 멘션

```
# 잘못된 사용
@channel 내일 회의 있어요 (업무시간 외 전체 알림 유발)

# 올바른 사용
@조인범 내일 2시 회의 확인 부탁드립니다
```

- `@channel` / `@here`: 긴급 공지에만 (업무시간 내)
- `@[이름]`: 특정인 확인 필요 시
- 일반 공유: 멘션 없이 게시

### 스레드(Thread) 필수 사용

메시지에 대한 답변은 반드시 스레드로 달기:

```
원본 메시지: "4/5 운영위 안건 공유합니다 (문서 링크)"
스레드 답변: "확인했습니다", "2번 안건 관련 자료 추가 첨부합니다"
```

채널 피드를 깨끗하게 유지하고 관련 대화를 한 곳에 모을 수 있습니다.

## 응답 시간 기대치

| 채널 유형 | 기대 응답 시간 |
|-----------|--------------|
| `#anchor-general` (질문) | 다음 업무일 이내 |
| 소속 본부 채널 | 당일 업무시간 내 |
| 개인 DM | 4시간 이내 (업무시간) |
| 긴급 표시 메시지 | 1시간 이내 |

- 업무시간 외 메시지는 다음날 업무시작 시 확인
- 장기 부재 시 Slack 상태 메시지 설정 권장

## AI 활용 공유 문화

`#anchor-ai-tips` 채널을 통해 AI 활용 노하우를 공유합니다.

### 공유 권장 내용

- Claude 스킬 사용 후기 및 결과물 스크린샷
- 유용한 프롬프트 (업무별 템플릿)
- AI가 잘 못하는 것 + 보완 방법
- 새로운 AI 도구 발견 시 소개

### 공유 형식 (권장)

```
[anchor-report] 개조식 변환 팁

상황: 5페이지 행사 결과 보고서를 2페이지로 압축해야 할 때

프롬프트:
"아래 내용을 교육부 제출용 개조식 보고서로 변환해줘.
 핵심 성과 수치는 반드시 포함하고, 분량은 A4 1페이지 이내로."

결과: [스크린샷]

팁: 분량 제한을 명시하면 훨씬 간결한 결과가 나옵니다
```

## 금지 사항

- 개인정보 (전화번호, 주민번호, 주소) 게시
- 급여·인건비 개인 내역 공유
- 미공개 예산 계획 또는 인사 정보 게시
- 타 기관·개인 비방
- 업무와 무관한 정치·종교 관련 내용

## 신규 구성원 온보딩

새로 합류한 구성원은:

1. `#anchor-general`에 자기소개 게시
   ```
   안녕하세요! [이름]입니다.
   [소속 본부]에서 [담당 업무]를 맡게 되었습니다.
   잘 부탁드립니다!
   ```

2. 소속 본부 채널에도 인사 게시
3. `#anchor-ai-tips` 구독 설정 (AI 활용 학습)
4. `anchor-tools` 설치 완료 후 `#anchor-ai-help`에 완료 인증 가능
